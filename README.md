# AIFChem OpenAI Shim

这个仓库现在只保留一条主线：

把 AIFChem 的真实网页聊天接口适配成 OpenAI `/v1/chat/completions`。

## 安装

```bash
npm install
```

如果要运行 Python 测试脚手架，先安装 `uv`：

```bash
python3 -m pip install --user uv -i https://pypi.org/simple
```

然后在仓库根目录初始化 Python 环境：

```bash
uv sync --dev
```

## 配置 `.env`

启动本地 shim 之前，先按下面步骤把 `.env` 填好。复制.env.example

推荐流程：

1. 登录 [chat.aifchem.com](https://chat.aifchem.com/)。
2. 打开浏览器开发者工具。
3. 进入 `Network` 面板。
4. 在聊天页实际发送一条消息。
5. 找到请求 `https://chat-app.aifchem.com/api/message/chat`。
6. 右键这个请求，选择 `Copy as cURL`。
7. 从 cURL 里提取：
   - `Authorization: Bearer ...` 对应 `AIFCHEM_AUTHORIZATION`
   - 请求体里的 `workflow_id` 对应 `AIFCHEM_WORKFLOW_ID`
8. 再找到请求 `https://chat-app.aifchem.com/api/thread/create`，或直接从 `api/message/chat` 的上下文里确认同一会话使用的 `conversation_id`，填到 `AIFCHEM_CONVERSATION_ID`。
9. 把它们写入本地 `.env`。

`.env` 示例：

```bash
AIFCHEM_AUTHORIZATION="Bearer <your-browser-jwt>"
AIFCHEM_CONVERSATION_ID="<your-conversation-id>"
AIFCHEM_WORKFLOW_ID="<your-workflow-id>"
```

注意：

- `.env` 已在 `.gitignore` 中，不要把真实值写回示例文件。

## 启动本地 shim

环境变量模板在 [.env.example](.env.example)，默认配置在 [examples/shim.config.example.json](examples/shim.config.example.json)。

```bash
set -a; source .env; set +a; npm run serve:shim
```

当前 shim 会：

- 先调用 `POST https://chat-app.aifchem.com/api/thread/create`
- 再调用 `POST https://chat-app.aifchem.com/api/message/chat`
- 对 `chat_app_message_api` 把完整 OpenAI `messages` 压成单个 `query`，而不再只取最后一条 `user`
- 可选按 `metadata.session_id` 复用同一个上游 thread/message 游标，支持多轮对话
- 默认过滤网页聊天里夹带的 UI 表单块和固定免责声明
- 把上游 SSE 转成 OpenAI 风格的非流式或流式 delta 输出

没有设置 `AIFCHEM_AUTHORIZATION` 时，进程会直接启动失败。

## 本地调用

非流式：

```bash
curl http://127.0.0.1:4010/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer sk-local-test' \
  -d '{
    "model": "aifchem-agent",
    "stream": false,
    "messages": [
      {"role": "user", "content": "hello"}
    ]
  }'
```

流式：

```bash
curl -N http://127.0.0.1:4010/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer sk-local-test' \
  -d '{
    "model": "aifchem-agent",
    "stream": true,
    "messages": [
      {"role": "user", "content": "hello"}
    ]
  }'
```

## 多轮会话

现在有两种方式让 shim 支持同一个 agent 的多轮对话：

- 客户端每一轮都发送完整的 OpenAI `messages` 历史。shim 会把完整历史序列化进上游 `query`，因此 assistant/tool 上下文不会丢。
- 客户端在 `metadata.session_id` 里传入一个稳定值。shim 会在服务端缓存这个 session 对应的 `thread_id / parent_message_id / workflow_id / conversation_id`，优先复用真实 chat-app thread。

推荐两种方式同时使用：既传完整 `messages`，也传 `metadata.session_id`。这样即使上游拒绝复用旧 thread，shim 也能新建 thread 后继续用完整历史重放上下文。

示例：

```bash
curl http://127.0.0.1:4010/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer sk-local-test' \
  -d '{
    "model": "aifchem-agent",
    "stream": false,
    "metadata": {
      "session_id": "demo-agent-1"
    },
    "messages": [
      {"role": "system", "content": "You are a careful chemistry assistant."},
      {"role": "user", "content": "I mixed reagent A."},
      {"role": "assistant", "content": "What solvent did you use?"},
      {"role": "tool", "content": "solvent=water"},
      {"role": "user", "content": "What should I do next?"}
    ]
  }'
```

`examples/python_chat_client.py` 现在会在 REPL 中自动保留完整消息历史，并默认生成一个 `session_id` 用于多轮测试。

## 输出过滤开关

`upstream.chatApp` 下现在支持两个可选开关，默认都为 `true`：

- `filterUiPayloads`: 过滤上游返回里的 ` ```data ...``` ` UI/表单块
- `filterDisclaimers`: 过滤固定免责声明 `This answer is generated by a Large Language Model, please verify the information independently.`

如果你想保留原始网页聊天输出，可以这样配置：

```json
{
  "upstream": {
    "chatApp": {
      "filterUiPayloads": false,
      "filterDisclaimers": false
    }
  }
}
```
